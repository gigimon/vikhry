# Step 32: RPS 1s в UI и синхронизация topbar

## Цель

Убрать рассинхрон RPS между таблицей и топбаром, перейти на расчет за последнюю секунду.

## Реализация

[x] Перейти на частоту обновления UI 1s (`REFRESH_INTERVAL_MS=1000`).
[x] Рассчитывать `RPS_1s` на фронтенде как дельту `aggregate_total.requests` между соседними тиками.
[x] Использовать `RPS_1s` в таблице статистики независимо от `Window/Whole test` режима.
[x] Рассчитывать topbar RPS как сумму `RPS_1s` по всем метрикам, кроме `source=step`.

## Прогресс

- [x] Таблица использует `RPS_1s`.
- [x] Topbar исключает step-метрики из суммарного RPS.
