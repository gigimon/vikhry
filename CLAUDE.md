# vikhry

## 1. Общее описание

**vikhry** — асинхронный распределённый фреймворк для нагрузочного тестирования,
ориентированный на высокую конкурентность, горизонтальное масштабирование
и поддержку глобальных уникальных ресурсов.

Проект предназначен для:

- нагрузочного тестирования HTTP / JSON-RPC / API систем
- тестирования распределённых сервисов
- сценариев с большим количеством уникальных пользователей/ресурсов
- выполнения нагрузки в кластере (несколько машин)
- онлайн-визуализации метрик

vikhry строится вокруг asyncio-модели и использует Redis как единый
координационный и событийный центр.

---

## 2. Основные принципы

- Полностью асинхронная архитектура
- Один воркер = один процесс = один event loop
- Множество VU (Virtual Users) на одном воркере
- Глобальные ресурсы через централизованный пул
- PREPARING → RUNNING модель выполнения
- Redis как единственная точка синхронизации
- Разделение backend (Python) и frontend (React)

## 4. Компоненты системы

### CLI
- запуск coordinator, worker
- изменение текущих настроек системы (например, target RPS)

### Coordinator (Robyn)

- REST API управления тестами
- WebSocket для live-метрик
- Оркестрация фаз PREPARING / RUNNING
- Provision scheduler
- Event aggregator

### Workers

- Выполнение VirtualUsers (async tasks)
- Создание ресурсов (в фазе PREPARING)
- Отправка событий в Redis

### Redis

Используется для:

- хранения состояния теста
- очередей provisioning jobs
- пула ресурсов
- lease-механизма
- event stream
- метрик (всех и агрегированных)

### Frontend (React)

- SPA интерфейс
- Управление тестами
- Графики RPS / latency / errors
- Мониторинг воркеров и ресурсов
